# Panorama Security Profile Group Auditor

A Python GUI application for auditing Security Profile Groups in Palo Alto Networks Panorama.

## Overview

This application allows network administrators to:

1. Select a Panorama instance from a dropdown list
2. Add or remove Panorama instances as needed
3. Specify a device group name (default is "any")
4. Select Security Profile Groups to audit
5. Choose whether to include disabled rules in the audit
6. Generate CLI commands for auditing these Security Profile Groups
7. Copy the generated commands to clipboard for use in Panorama CLI

## Requirements

- Python 3.6 or higher
- Tkinter (included with most Python installations)

## Installation

1. Ensure Python is installed on your system
2. Clone or download this repository
3. No additional packages are required as the application uses only standard libraries

## Usage

1. Run the application:
   ```
   python pan_audit_gui.py
   ```

2. Using the application:
   - **Add Panorama Instances**: Click the "Add" button to add a new Panorama hostname or IP address
   - **Select Panorama**: Choose a Panorama instance from the dropdown
   - **Load Panorama**: Click the "Load" button to load the configuration for the selected Panorama
   - **Save Panorama**: Click the "Save" button to save the current Panorama configuration
   - **Device Group**: Enter a device group name or leave as "any" to search across all device groups
   - **Set Device Group**: Click the "Set" button to save the device group for the current Panorama
   - **Include Disabled Rules in Results**: Check this box if you want to include disabled rules in the audit results
   - **Security Profile Groups**: Add Security Profile Groups using the Plus button
   - **Remove Security Profile Groups**: Select one or more SPGs (use CTRL+click for multiple selection) and click the Minus button to remove them
   - **Generate Commands**: Click the "Generate CLI Commands" button to create the audit commands
   - **Copy All Commands**: Use the "Copy All Commands" button to copy all commands for use in Panorama CLI
   - **Execute All Commands**: Click the "Execute All Commands" button to run all commands and track their progress
   - **Analyze Output**: Click the "Analyze Output" button to consolidate all output files into a single Excel workbook
   - **Clear Output Files**: Click the "Clear Output Files" button to clear the list of output files
   - **Validate PHP**: Click the "Validate PHP" button to check if PHP is installed and working correctly
   - **Individual Command Sections**: Each command is displayed in its own section with:
     - A section title
     - The command text
     - A "Copy to Clipboard" button to copy just that command
     - An "Execute Command" button to run the command in the panPHP/utils directory

## Configuration

The application stores its configuration in JSON files in the `config` directory. The main configuration file is `config/panAudit.json`, and the audit definitions are stored in `config/audits.json`. This configuration includes:

1. A list of Panorama instances, each with its own:
   - Device group setting
   - List of Security Profile Groups to audit (specific to each Panorama)
2. The currently selected Panorama instance
3. Whether to include disabled rules in the audit

This allows all settings to persist between sessions.

## Generated Commands

The application generates CLI commands that use the pan-os-php.php script to export security rules to Excel files. Each command is displayed in its own section with a title, the command text, a "Copy to Clipboard" button, and an "Execute Command" button.

## Output Analysis

The application provides functionality to analyze the output files generated by the audit commands:

### Analyze Output

When you click the "Analyze Output" button:

1. The application consolidates all output files into a single Excel workbook
2. Each output file becomes a separate worksheet in the workbook
3. Column widths are automatically adjusted based on content
4. The consolidated workbook is saved to a location of your choice
5. A summary window displays statistics about the audit results

This feature requires additional Python packages:
- BeautifulSoup4 (`pip install beautifulsoup4`)
- pandas (`pip install pandas`)
- xlsxwriter (`pip install xlsxwriter`)

If these packages are not installed, the application will display an error message with installation instructions.

### Clear Output Files

The "Clear Output Files" button allows you to clear the list of output files that have been generated. This is useful if you want to start a new analysis with a fresh set of output files.

### Section 0: All Rules Report
- A command to report all rules found in the Panorama (exported to "0_AllRules.xls")

### Section 1: Logging
- 1.1: A command to report rules that don't have logging enabled (exported to "1.1_NoLog.xls")
- 1.2: A command to report rules that have logging at the start (exported to "1.2_LogStart.xls")
- 1.3: A command to report rules that have logging at the end (exported to "1.3_LogEnd.xls")
- 1.4: A command to report rules that have logging enabled but no log forwarding profile (exported to "1.4_Unforwarded.xls")

### Section 2: Security Profiles
- 2.1: A command to report rules that have no security profile but allow traffic (exported to "2.1_NoSecProf.xls")
- 2.2: A command to report rules that use individual security profiles instead of profile groups (exported to "2.2_SecProfNotGroup.xls")
- 2.3: A command to report rules that have security profiles but deny traffic (exported to "2.3_SecProfDeny.xls")

### Section 3: Application and Service
- 3.1: A command to report rules that allow any application and any service (exported to "3.1_AllowAnyAppAnyPort.xls")
- 3.2: A command to report rules that allow any application (exported to "3.2_AllowAnyApp.xls")
- 3.3: A command to report rules that allow any service (exported to "3.3_AllowAnyPort.xls")
- 3.4: A command to report rules that deny traffic but specify a service (exported to "3.4_DenyNonAnyPort.xls")
- 3.5: A command to report rules that allow traffic with any source and any destination (exported to "3.5_AnySourceAnyDest.xls")
- 3.6: A command to report rules that allow traffic from any zone and any source (exported to "3.6_AnyAnySource.xls")
- 3.7: A command to report rules that allow traffic from any source IP (exported to "3.7_AnyIPSource.xls")
- 3.8: A command to report rules that allow traffic to any zone and any destination (exported to "3.8_AnyAnyDest.xls")
- 3.9: A command to report rules that allow traffic to any destination IP (exported to "3.9_AnyIPDest.xls")

### Section 4: Security Profile Groups
- For each Security Profile Group, a command that exports rules to an Excel file with a numbered prefix (4.1, 4.2, etc.) and a filter for that SPG

All commands include the "shadow-ignoreinvalidaddressobjects" argument to ignore invalid address objects.

If "Include disabled rules" is unchecked, the commands will include a filter to exclude disabled rules.

### Command Execution

When you click the "Execute Command" button for a specific command:
1. The appropriate shell will be opened based on your operating system (PowerShell for Windows, Terminal for Mac)
2. The shell will automatically navigate to the panPHP/utils directory
3. The command will be executed in the shell
4. You can see the output directly in the shell window
5. The shell will wait for you to press Enter before closing, allowing you to review the output

Example:
```
# Commands for Panorama: panorama.example.com
# Device Group: any
# Include Disabled Rules: No

# All Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:0_AllRules.xls filter="!(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# Unlogged Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:Unlogged_Rules.xls filter="!(log at.start) and !(log at.end) and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# LogStart Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:1.2_LogStart.xls filter="log at.start and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# LogEnd Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:1.3_LogEnd.xls filter="log at.end and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# Unforwarded Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:1.4_Unforwarded.xls filter="((log at.start) or (log at.end)) and !(logprof is.set) and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# NoSecProf Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:2.1_NoSecProf.xls filter="secprof not.set and action is.allow and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# SecProfNotGroup Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:2.2_SecProfNotGroup.xls filter="secprof type.is.profile and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# SecProfDeny Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:2.3_SecProfDeny.xls filter="action is.deny and secprof is.set and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# AllowAnyAppAnyPort Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:3.1_AllowAnyAppAnyPort.xls filter="(action is.allow) and (app is.any) and (service is.any) and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# AllowAnyPort Rules Report

# DenyNonAnyPort Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:3.4_DenyNonAnyPort.xls filter="(action is.deny) and !(service is.any) and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# AnySourceAnyDest Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:3.5_AnySourceAnyDest.xls filter="(action is.allow) and (dst is.any) and (src is.any) and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# AnyAnySource Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:3.6_AnyAnySource.xls filter="(action is.allow) and (from is.any) and (src is.any) and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# AnyIPSource Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:3.7_AnyIPSource.xls filter="(action is.allow) and (src is.any) and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# AnyAnyDest Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:3.8_AnyAnyDest.xls filter="(action is.allow) and (to is.any) and (dst is.any) and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# AnyIPDest Rules Report
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:3.9_AnyIPDest.xls filter="(action is.allow) and (dst is.any) and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

# Section 4: Security Profile Groups

#	4.1 - Block-High-Risk
php pan-os-php.php type=rule in=api://panorama.example.com ruletype=Security location=any actions=exporttoexcel:4.1_Block-High-Risk.xls filter="secprof group.is Block-High-Risk and !(rule is.disabled)" shadow-ignoreInvalidAddressobjects

```

## Customization

The generated commands can be customized by modifying the `generate_commands` method in the `PanoramaAuditGUI` class.
